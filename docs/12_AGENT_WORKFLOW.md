# 12 Agent Workflow

## Purpose

Define how agents should work in this repository.

## Workflow

1. Read current alignment and milestone context.
2. Check for rework or audit reports.
3. Keep scope limited to the approved task.
4. Avoid implementing business code during documentation tasks.
5. Verify file existence and Git state before completion.
6. Report created files, commit hash, and next steps.

## Commit Discipline

Commits should describe the actual scope of the change.

## Bootstrap Constraint

Agent work during Bootstrap is limited to documentation and directory scaffolding.
