# 02 TDD

## Technical Direction

The system will be built as a web application with a backend API, frontend reader, content ingestion pipeline, grounded retrieval layer, learning state, Zotero integration, and knowledge graph.

## Planned Components

- Backend API service.
- Frontend web application.
- Source pipeline.
- Article storage.
- Search and retrieval.
- RAG orchestration.
- Learning state and history.
- Zotero bridge.
- Knowledge graph.

## Bootstrap Constraints

This document defines direction only. No component is implemented during Bootstrap.

## Future Technical Decisions

Technical decisions should be recorded in `ADR/` before implementation when they affect architecture, storage, AI providers, retrieval, or deployment.
