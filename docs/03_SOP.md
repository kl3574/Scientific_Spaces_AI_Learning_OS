# 03 SOP

## Operating Principles

- Define scope before implementation.
- Keep sources traceable.
- Keep AI outputs grounded in article, paper, and section references.
- Prefer small, verifiable milestones.

## Standard Workflow

1. Update or read the relevant milestone.
2. Create an implementation plan.
3. Implement only the approved scope.
4. Add tests matching the risk level.
5. Verify locally.
6. Commit with a clear message.
7. Push to `origin/main` or a task branch as appropriate.

## Bootstrap Rule

Bootstrap may create documentation and empty directories only. It must not implement business logic.
